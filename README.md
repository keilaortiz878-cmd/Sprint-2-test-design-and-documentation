Sprint 2: QA Testing Fundamentals

## 📋 Project Overview
This repository documents my learning journey through QA Testing Fundamentals, where I gained comprehensive knowledge of Quality Assurance principles and practices. The sprint followed an engaging narrative featuring Ali, a QA expert investigating technical issues in the fictional town of Codetopia.

## 🎯 Learning Objectives Achieved
- Understanding the QA engineer role and responsibilities
- Mastering Software Development Life Cycle (SDLC) concepts
- Learning various testing types and methodologies
- Gaining hands-on experience with industry-standard tools
- Developing professional bug reporting skills

## 📚 Key Topics Covered

### QA Fundamentals
- Role of QA engineers in development teams
- Hard and soft skills required for QA professionals
- Quality assurance standards and documentation

### Software Development Life Cycle (SDLC)
- Sequential vs. iterative development methodologies
- Waterfall Model vs. Agile and Scrum approaches
- QA involvement at every SDLC stage

### Testing Methodologies
- **Automation Levels**: Manual vs. Automated testing
- **Functional Categories**: Functional vs. Non-functional testing
- **Testing Iterations**: Regression vs. New feature testing
- **Coverage Depth**: Smoke, Extended, and Exhaustive testing
- **Isolation Levels**: Component, Integration, and System testing
- **Planning Approaches**: Scripted vs. Exploratory testing
- **Code Access**: Black-box, White-box, and Gray-box testing

### Exploratory Testing Techniques
- Tour testing (feature, variability, complexity tours)
- Session-based testing approaches

## 🛠️ Tools and Technologies
